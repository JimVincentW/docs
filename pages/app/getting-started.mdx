# Getting Started

Welcome to Open Politics HQ! This guide will get you up and running quickly.

## Choose Your Setup

Select the setup method that best fits your needs:

<Tabs>
<Tab title="Hosted Instance (Recommended)">
The fastest way to get started. We handle the infrastructure, you focus on analysis.

<Steps>
<Step title="Register">
Sign up at [open-politics.org](https://open-politics.org)
</Step>
<Step title="Add API Keys">
Configure your LLM API keys (OpenAI, Google, Anthropic, or Ollama)
</Step>
<Step title="Start Analysing">
Upload content and create your first schema
</Step>
</Steps>
</Tab>

<Tab title="Self-Hosted Setup">
For privacy, customization, or institutional requirements. Run everything on your own infrastructure.

<Steps>
<Step title="Prerequisites">
Docker, Git, 32GB RAM recommended
</Step>
<Step title="Clone & Setup">
```bash
git clone https://github.com/open-politics/open-politics-hq.git
cd open-politics-hq
cp .env.example .env
# Edit .env with your configuration
```
</Step>
<Step title="Start Services">
```bash
docker compose up
```
</Step>
<Step title="Access Platform">
Open http://localhost:3000
</Step>
</Steps>

<Warning>
**Security Notice:** Change the default admin credentials immediately:
- Username: `app_user`
- Password: `app_user_password`
</Warning>
</Tab>
</Tabs>

## Your First Analysis

<Steps>
<Step title="Upload Content">
Upload PDFs, web articles, CSV files, or text documents
</Step>
<Step title="Create Schema">
Define what to extract in plain English:

```json
{
  "topic": {
    "type": "string",
    "description": "Main topic"
  },
  "sentiment": {
    "type": "string",
    "description": "Sentiment: [positive, negative, neutral]"
  },
  "entities": {
    "type": "array",
    "description": "Key people, organisations, locations"
  }
}
```
</Step>
<Step title="Run Analysis">
Select content and schema, choose AI model, then run analysis
</Step>
<Step title="Explore Results">
View results in tables, charts, maps, or export data
</Step>
</Steps>

## Supported Content

**Formats:** PDFs, web articles, text files, CSV files  
**AI Models:** OpenAI GPT-4/3.5, Google Gemini, Anthropic Claude, Ollama (local)
